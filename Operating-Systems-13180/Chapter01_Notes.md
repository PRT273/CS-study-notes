第一章 操作系统概论 （Introduction to Operating Systems）



1.1 什么是操作系统：



操作系统（Operating System, OS） 是一种系统软件，

用于管理计算机硬件资源，并为用户程序提供统一、抽象、可控的运行环境。



从功能角度看，操作系统位于：

硬件（Hardware） ← OS ← 应用程序（Applications） ← 用户（User）



操作系统的核心作用可以概括为三点：



资源管理者：

管理 CPU、内存、磁盘、I/O 设备等有限资源。



控制程序：

控制程序的执行，防止错误和资源滥用。



抽象提供者：

将复杂硬件抽象为易用的接口（如进程、文件、虚拟内存）。



1.2 操作系统的目标：



（1）方便性（Convenience）

操作系统通过抽象机制，使用户无需关心底层硬件细节即可使用计算机。



例如：

文件系统 → 屏蔽磁盘结构

进程 → 屏蔽 CPU 调度细节



（2）有效性（Efficiency）

在保证正确性的前提下，提高系统资源的利用率：

提高 CPU 使用率

减少 I/O 等待时间

提升系统吞吐量



（3）可扩展性（Extensibility）

操作系统应支持：

新硬件

新设备驱动

新系统调用





1.3 操作系统的主要功能



处理机管理

主要负责：

进程的创建与撤销

进程调度（CPU Scheduling）

进程同步与互斥

死锁处理



关键概念：

进程（Process）

线程（Thread）

上下文切换（Context Switch）





存储器管理



主要功能：

内存分配与回收

地址映射

内存保护

虚拟内存管理



典型机制：

分页、分段和虚拟内存





文件管理



操作系统通过文件系统实现：

文件的创建、删除、读写

目录管理

存储空间分配

文件访问权限控制



核心抽象：

文件（File）

目录（Directory）

inode（在类 Unix 系统中）





设备管理



功能包括：

设备分配与回收

I/O 调度

缓冲管理

中断处理



设备通常被抽象为：

字符设备

块设备





1.4 操作系统的发展历程（简述）



（1）手工操作阶段



无操作系统

程序直接运行在硬件上



（2）批处理系统



提高 CPU 利用率

无交互能力



（3）多道程序系统



多个程序同时驻留内存

提高资源利用率



（4）分时系统



面向交互

响应时间短



（5）现代操作系统



多用户、多任务

支持网络、虚拟化、容器





1.5 操作系统的基本类型



批处理操作系统



特点：

成批处理作业

无用户交互



分时操作系统



特点：

多用户共享系统

时间片轮转



实时操作系统（RTOS）



特点：

强调响应时间的确定性

用于工业控制、嵌入式系统



分布式操作系统



特点：

多台计算机协同工作

对用户呈现为一个整体系统





1.6 操作系统的结构（概览）



（1）单体内核（Monolithic Kernel）

所有功能在内核态

性能高，可靠性相对低



（2）微内核（Microkernel）

内核最小化

高模块化，通信开销大



（3）分层结构 / 模块化结构

易维护

工程实践常用





1.7 本课程的学习目标



通过学习操作系统，应达到：

理解操作系统的基本原理

掌握进程、内存、文件等核心机制

为系统编程、后端开发、机器学习实验环境提供理论基础
